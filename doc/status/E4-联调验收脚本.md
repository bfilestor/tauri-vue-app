# E4 多用户多成员隔离 · 联调验收脚本

## 1. 目标

验证 V2 多用户多成员模型在桌面端实际交互下满足以下要求：

- 注册或首次登录后，系统自动创建默认“本人”成员。
- 用户可新增家庭成员并切换当前成员。
- 上传、OCR、AI 分析、趋势、AI 问答均严格按当前成员隔离。
- 不同成员之间的报告、OCR 结果、AI 分析、趋势数据、聊天记录互不影响。

## 2. 验收前准备

### 2.1 环境

- 桌面端为当前本地分支最新代码。
- 本地数据库为新初始化环境，或手动删除旧测试数据库后再启动。
- 如需联调服务端成员接口，确保 `/app-api/family-members` 系列接口可用。

### 2.2 建议清理项

- 删除本地应用数据目录下旧版 `health_guard.db`。
- 删除本地 `pictures/` 测试图片目录。
- 准备两组可区分的测试报告图片：
  - A 成员报告：至少 1 张
  - B 成员报告：至少 1 张

### 2.3 测试角色

- 业务用户：`U1`
- 默认成员：`M1-本人`
- 第二成员：`M2-家人`

## 3. 核心验收路径

### 场景 1：注册/首登自动创建本人成员

前置条件：
- 本地无该账号历史会话
- 服务端该账号无家庭成员，或返回空成员列表

步骤：
1. 启动应用
2. 注册新账号或首次登录现有账号
3. 等待账户上下文初始化完成
4. 打开设置页或侧边栏成员区域

预期结果：
- 系统自动创建 `relationCode=SELF` 的默认成员
- 当前成员自动指向“本人”
- 上传、历史、趋势、AI 问答页面可正常进入，不再处于成员阻塞状态

### 场景 2：新增第二个家庭成员并切换

前置条件：
- 已完成场景 1

步骤：
1. 进入设置页成员管理
2. 新增一个家庭成员，例如“母亲”
3. 保存后切换当前成员为“母亲”
4. 依次打开上传页、历史页、趋势页、AI 问答页

预期结果：
- 成员列表出现“本人”和“母亲”
- 当前成员切换成功并持久化
- 各业务页跟随当前成员刷新
- 新成员初始为空数据状态，不显示“本人”的历史数据

### 场景 3：本人上传报告并完成 OCR/AI 分析

前置条件：
- 当前成员切回“本人”

步骤：
1. 在上传页为“本人”创建一条检查记录
2. 上传 A 成员测试报告
3. 执行 OCR
4. OCR 完成后查看 OCR 结果
5. 执行 AI 分析
6. 在历史页和趋势页查看结果

预期结果：
- 上传记录只出现在“本人”名下
- OCR 结果只挂在“本人”的记录下
- AI 分析结果只出现在“本人”的历史记录和 AI 历史中
- 趋势页仅基于“本人”的指标值出图

### 场景 4：家人上传报告并验证与本人隔离

前置条件：
- 已完成场景 3
- 当前成员切换为“母亲”

步骤：
1. 为“母亲”创建一条检查记录
2. 上传 B 成员测试报告
3. 执行 OCR 和 AI 分析
4. 查看“母亲”的上传、历史、趋势、AI 历史
5. 再切回“本人”重复查看同样页面

预期结果：
- “母亲”只能看到自己的记录、OCR、AI 分析和趋势
- 切回“本人”后只能看到“本人”的数据
- 即使同一天上传相同项目，也不会混出对方数据

### 场景 5：AI 问答会话按成员独立

前置条件：
- 已完成场景 3 和场景 4

步骤：
1. 切到“本人”，在 AI 问答中连续提 2 个问题
2. 记录当前聊天历史内容
3. 切到“母亲”，在 AI 问答中提 1 个完全不同的问题
4. 再切回“本人”

预期结果：
- “母亲”进入 AI 问答时，不显示“本人”的聊天记录
- 切回“本人”后，仍能看到“本人”之前的独立对话
- 两个成员聊天记录互不串话

### 场景 6：清空当前成员聊天记录不影响其他成员

前置条件：
- “本人”和“母亲”都至少有 1 轮 AI 对话

步骤：
1. 切到“本人”
2. 执行清空聊天历史
3. 确认“本人”聊天列表为空
4. 切到“母亲”

预期结果：
- “本人”聊天记录被清空
- “母亲”聊天记录仍然保留
- 切回“本人”时不会出现“母亲”对话

### 场景 7：删除当前成员记录或文件不影响其他成员

前置条件：
- “本人”和“母亲”都已有各自报告数据

步骤：
1. 切到“本人”
2. 删除“本人”某条检查记录，或删除某个文件
3. 查看“本人”历史
4. 切到“母亲”查看其历史、文件、OCR、趋势

预期结果：
- 删除操作只作用于当前成员
- 被删记录关联的 OCR/AI/指标值同步移除
- 其他成员数据完整保留

### 场景 8：重启应用后恢复当前成员上下文

前置条件：
- 当前成员为“母亲”

步骤：
1. 关闭应用
2. 重新启动应用
3. 等待会话恢复与账户上下文加载
4. 打开上传页、历史页、AI 问答页

预期结果：
- 当前成员优先恢复为上次选择的“母亲”
- 页面直接进入“母亲”的数据上下文
- 不会自动回退成“本人”或出现上下文丢失

## 4. 边界验收

### 边界 1：成员上下文缺失时业务阻断

步骤：
1. 人为制造成员上下文缺失场景
2. 尝试进入上传、OCR、AI 分析、AI 问答

预期结果：
- 前端统一阻断操作
- 有明确提示要求先选择或初始化成员

### 边界 2：重复登录已有本人成员

步骤：
1. 使用已有成员数据账号重新登录

预期结果：
- 不重复创建新的 `SELF` 成员
- 直接恢复已有成员上下文

### 边界 3：删除最后一个成员

步骤：
1. 只保留 1 个成员时尝试删除

预期结果：
- 阻止删除
- 明确提示至少保留 1 个成员
